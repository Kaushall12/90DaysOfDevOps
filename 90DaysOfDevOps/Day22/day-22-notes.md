#### Notes – Git Basics

##### 

##### 1\. Difference between git add and git commit



git add moves changes to staging area.

git commit permanently records staged changes in repository history.



---



##### 2\. What is the staging area?



The staging area is a preparation zone.

It allows selective commits instead of committing everything at once.



---



##### 3\. What does git log show?



Commit ID (hash)

Author

Date

Commit message



---



##### 4\. What is the .git folder?



It contains:

\- Commit history

\- Objects database

\- Branch references

\- Configuration



Deleting it removes all version control history.



---



##### 5\. Working directory vs staging vs repository



Working directory → Files you edit

Staging area → Changes prepared for commit

Repository → Saved commit history
